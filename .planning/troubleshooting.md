## 확인된 이슈
* 조치된 내용은 체크박스를 추가하여 마킹할 것. 대괄호 안에 확인 필요라고 되어있으면 기존에 조치했으나 여전히 문제가 있는 경우라고 사용자가 확인한 내용임.

## 기능 추가/개선

### 파일 탐색 기능 추가
1) 좌측 사이드바 추가
- [ ] 드라이브(C:, D: 등) 부터 시작해서 트리 구조로 펼칠 수 있게 구현
- [ ] 사이드바에서 폴더를 클릭 시 시각적으로 펼쳐지면서 하위 폴더들이 나타남(펼쳐졌을 때 폴더 아이콘 옆에 'v' 아이콘 표시, 접혀있을 때 '>')
- [ ] 폴더를 더블 클릭 시에는 가운데 메인 패널(검색 결과 창)에 해당 폴더 내 파일 목록 표시
    - 파일 목록은 기존에 검색창에서 검색 시 나타는 결과와 동일한 포맷으로 표시(파일 아이콘, 파일이름, 파일 경로, 수정일)
        - 필요하면 Model 클래스 등을 사용하여 메인 패널에 파일목록을 표시하는 로직을 추상화하고 향후 메인 패널에 파일 목록을 표시하는 기준 인터페이스로 사용
    - DB에 등록된 파일이라면 저장된 태그도 표시되어야 함.
    - DB에 등록되지 않은 파일이라면 태그 표시 대신 '미등록 파일' 이라고 표시되어야 함.

2) 파일 경로 창 추가
- [ ] 메인 패널과 보기모드 아이콘 사이에 배치
- [ ] 좌측 사이드 바에서 폴더 더블 클릭 시 해당 폴더 경로가 표시되어야 함.
- [ ] 경로 표시줄의 각 경로 부분(드라이브명, 폴더명 등)을 클릭하면 해당 경로로 이동 가능해야 함.
- [ ] 경로 표시줄에 수동으로 직접 폴더 경로를 입력한 경우 해당 폴더 내 파일 목록을 메인 패널에 표시해야 함.
    - 폴더 경로가 아니라 파일 경로인 경우에는 해당 파일이 속한 폴더의 파일 목록을 메인 패널에 표시해야함.
- [ ] 사이드바를 통해 이동한 것이 아니라 검색창을 통해 검색을 한 경우에는 '{검색어} 검색 결과'와 같이 표시되어야 함.
- [ ] 경로 표시줄의 오른쪽 끝에 '새로고침' 아이콘을 추가하고 클릭 시 현재 폴더의 파일 목록을 새로고침해야 함.
