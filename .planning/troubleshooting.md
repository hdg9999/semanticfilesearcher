## 버그

- [x] 태그 창에서도 엔터키 입력시 검색이 실행되어야함. 전에 됐었는데 어느시점부터 안됨.
    - 검색어 입력창에 input이 없이 태그 입력(GUI적으로 보았을때 태그 칩 아이템이 1개 이상 있음)만 있는 상태에서 태그창에서 엔터키를 입력하면 벡터 검색은 수행하지 않고 RDB에서 태그만 검색해야함.
        - [x] 검색 자체는 잘 수행되는데, 원래 검색 결과에 나타나던 각 파일 우측에 표시되던 각 파일의 태그 칩 항목이 보이지 않음. 기존 검색 결과와 동일하게 태그도 표시하도록 수정 필요.
    - 검색어 입력창 input도 있고, 태그 입력(GUI적으로 보았을때 태그 칩 아이템이 1개 이상 있음)도 있는 상태에서 태그창에서 엔터키를 입력하면 기존 로직대로 벡터검색 및 태그 필터링 수행하면 됨.
    - 검색어 입력창에도 input이 없고, 태그 입력창에도 태그 칩이 없는 경우(태그 입력창에 텍스트만 있는 경우는 태그 입력이 없는 것으로 간주함) 엔터키를 입력해도 아무 작업도 수행 안해야 함.
    
- [x] 태그 입력시 자동완성 리스트에서 태그를 선택한 경우 해당 텍스트가 태그 칩으로 변환이 되고 텍스트는 사라져야하는데, 태그칩 생성 후 텍스트도 그대로 입력창에 남아있어서 중복 출력된 상태가 됨.
